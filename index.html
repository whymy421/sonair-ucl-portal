Lab Identity

The UCL Robotics Lab focuses on remote teleoperation, sim2real transfer, and networked robot control. Our team investigates how network latency and jitter affect the fidelity of robot commands transmitted between simulation environments and physical hardware.

We are open to collaboration on federated benchmarking, cross-institutional teleoperation experiments, and shared dataset curation for the SONAIR network.

Interpretation

Network Stability: The mean RTT of ~55 ms with a range of 14–401 ms indicates generally responsive network conditions, but with significant outlier spikes. The P95 RTT reveals that 5% of measurements exceed a much higher threshold, suggesting intermittent congestion or routing variability on the UCL–Nottingham link.

Command Latency: The command latency histogram clusters tightly around 20–40 ms for the majority of commands, which is acceptable for non-safety-critical teleoperation. However, a long tail extending beyond 100 ms means that some robot commands experience delays that could cause visible desynchronisation between the operator's input and the robot's physical response.

Sim2Real Implication: In simulation, commands execute instantaneously with zero network delay. In the real teleoperation setup, the variable latency introduces a timing gap — the sim2real gap. A robot trajectory planned in simulation at 50 Hz assumes each command arrives within 20 ms, but the real network delivers commands with a standard deviation of ~25 ms. This means the physical robot's trajectory will diverge from the simulated one, particularly during high-speed motions or precision tasks.

Session Events: The session audit log shows a burst of link_degraded rejections near the end of the session (around t=420s), correlating with RTT spikes visible in the time-series plot. This confirms that network instability directly impacts teleoperation reliability and contributes to the sim2real gap.

Challenge 1 — Sim2Real Gap Metric

A novel metric to quantify the simulation-to-real timing gap for networked teleoperation

Temporal Fidelity Index (TFI)

We propose the Temporal Fidelity Index (TFI) — a composite metric that captures how faithfully a real networked teleoperation system reproduces the timing assumptions of its simulation counterpart.

Definition

TFI = (1 - D_norm) × (1 - J_norm) × (1 - L_rate)

Where:

  • D_norm = normalised mean excess delay = mean(RTT - RTT_sim) / RTT_max, where RTT_sim = 0 ms (ideal simulation)
  • J_norm = normalised jitter = std(RTT) / RTT_max, capturing timing unpredictability
  • L_rate = packet loss rate = lost_packets / total_packets

Why It Matters

Existing sim2real metrics focus on positional or visual accuracy. TFI specifically targets the temporal dimension — the gap between when a command should arrive (simulation assumption) and when it actually arrives (real network). For teleoperation, timing fidelity is often more critical than positional accuracy because delayed commands accumulate into trajectory divergence.

How to Adopt Across SONAIR

TFI can be computed for any teleoperation session that logs RTT and packet loss. Each campus node in the SONAIR network could publish its TFI alongside raw data, enabling cross-institutional comparison: which network links and configurations yield higher temporal fidelity? Over time, a TFI leaderboard could drive improvements in teleoperation infrastructure, identify bottleneck links, and establish minimum fidelity thresholds for safety-critical applications.
